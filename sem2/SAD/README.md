# 🎯 Course Overview

System Analysis and Design (SAD) focuses on the processes, methodologies, and techniques used to analyze business requirements and design effective information systems. The course introduces students to the System Development Life Cycle (SDLC), requirements gathering, process modeling, system design, and project documentation.

Through group projects and case studies, students learn how to transform business problems into structured system solutions.

---

# 📂 Project and Learning Activities

## 📋 System Analysis Project

### Description

The project involved analyzing a real-world problem, identifying stakeholder requirements, and proposing a system solution. Various analysis and design techniques were applied to model system processes, data flows, and user interactions.

### Key Learning Areas

- Requirements Gathering
- Stakeholder Analysis
- Business Process Analysis
- Problem Identification
- Solution Planning

### Reflection

This project taught me that understanding user requirements is one of the most important stages in system development. A well-designed system begins with a thorough understanding of the problem and user needs.

### Key Learning Areas

## 📊 System Modeling

### Description

Developed diagrams and models to represent system requirements and processes.

### Key Learning Areas

- Use Case Diagrams
- Activity Diagrams
- Data Flow Diagrams (DFD)
- Entity Relationship Diagrams (ERD)
- Process Modeling

### Reflection

System modeling improved my ability to visualize how users, processes, and data interact within a system. These diagrams helped transform abstract requirements into clear system designs.
