# 🎯 Course Overview

Human Computer Interaction (HCI) focuses on the design, evaluation, and implementation of interactive systems that provide meaningful and efficient user experiences. The course explores user-centered design principles, usability, accessibility, interface design, prototyping, and user evaluation techniques.

Throughout the semester, our group developed **BioFlora**, a web-based platform, and produced a comprehensive project report that applied HCI principles to solve real-world user needs.

---

# 📂 Project and Learning Activities

## 🌱 BioFlora Website Project

### Description

BioFlora is a web-based platform designed to provide users with information and guidance related to plants and flora. The project focused on creating an intuitive, user-friendly, and visually appealing interface while ensuring accessibility and usability.

### Key Learning Areas

- User-Centered Design (UCD)
- User Interface (UI) Design
- User Experience (UX) Principles
- Wireframing and Prototyping
- Website Development
- Accessibility Considerations
- Usability Evaluation

### Reflection

Developing BioFlora taught me that successful systems are not only functional but also easy and enjoyable to use. Throughout the design process, our team continuously considered user needs, navigation flow, visual hierarchy, and accessibility. This project improved my understanding of how design decisions directly influence user satisfaction and system effectiveness.

# 💡 Overall Reflection

Human Computer Interaction provided valuable insights into the relationship between technology and users. Before taking this course, I often focused primarily on functionality when developing systems. However, HCI taught me that usability, accessibility, and user satisfaction are equally important factors in system success.

The BioFlora project allowed me to apply theoretical concepts in a practical setting. Through designing interfaces, evaluating usability, and improving user experiences, I gained a deeper appreciation for user-centered design methodologies.

As a future Data Engineer, understanding user needs remains important because data-driven systems and dashboards must be designed in ways that users can easily understand and interact with.
